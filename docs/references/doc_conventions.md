# Doc Conventions

Last Updated: 2026-04-09

## Rules
- Every edited Markdown file includes `Last Updated`.
- Use current-state docs to describe how the system works now.
- Put generated inventories and matrices in `docs/generated/`.
- Put temporary launch or execution details in `docs/exec-plans/active/`.
- Avoid duplicating the same fact in README, design docs, and exec plans.

## Status Vocabulary
- Severity:
  - `P0 launch blocker`
  - `P1 must-fix`
  - `P2 should-fix`
  - `P3 defer`
- Launch status:
  - `ready`
  - `fix-before-launch`
  - `ship-with-guard`
  - `defer`
